# Copyright & Content Guidelines

## Legal Protection
- **Copyright Notice:** "Â© [Year] [Brand Name]. All rights reserved."
- **Documents:** Include Terms of Service and Privacy Policy links in footer.
- **Open Source:** Clearly state dependencies (MIT, Apache 2.0).
- **User Content:** ToS establishes ownership and usage rights.
- **Third-party Assets:** Maintain attribution file for icons, fonts, images.

## Attribution Requirements
- **Fonts:** Google Fonts (OFL), paid fonts (commercial license).
- **Icons:** Credit in README if required (Font Awesome, Ionicons).
- **Images:** Unsplash/Pexels require creator attribution.
- **Libraries:** Preserve copyright headers in source files.
- **AI Content:** Disclose when using AI-assisted design/copy.

## Content Ownership
- **Original Designs:** Automatically copyrighted upon creation.
- **Trademarks:** Register trademarks for logos and names.
- **Code:** Add LICENSE file (MIT recommended for flexibility).
- **Data:** GDPR/CCPA compliance for data collection.
- **API:** Creative Commons license for public APIs.

## Best Practices
- **Mockups:** Watermark design mockups shared publicly.
- **Version Control:** Git with clear commit history as proof of authorship.
- **Documentation:** README with credits section.
- **Contributors:** Recognize team member contributions.
- **Audits:** Regular review of dependencies for license compatibility.
